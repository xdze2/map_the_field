## Completed

### 1. Script Refactoring & Data Pipeline
- **Split single script into two:**
  - `download_entreprises.py` — Downloads all pages with automatic pagination, respects rate limits (7 req/s), saves to JSONL
  - `view_entreprises.py` — Views downloaded data offline, supports single file or directory input
- **Key features:**
  - JSONL format (one enterprise per line) for streaming/incremental processing
  - Metadata logging to `search_log.jsonl` (timestamp, postal code, filters, total results, status)
  - Condensed view (default): name, age, activity, size, directors
  - Full view: all available details with finances, addresses, labels
  - JSON output for programmatic use

### 2. Configuration Management

### 3. Directory Support

## Architecture Overview

```
Raw Data Collection
├── download_entreprises.py
│   ├── Fetches from API Recherche d'Entreprises
│   ├── Handles pagination automatically
│   ├── Respects rate limits (7 req/s)
│   └── Saves to JSONL + metadata log
│
├── data/raw/searches/*.jsonl
│   └── One enterprise per line (full API response)
│
└── data/raw/metadata/search_log.jsonl
    └── Search parameters & results count

Data Exploration
├── view_entreprises.py
│   ├── Reads JSONL (offline, no API calls)
│   ├── Supports file or directory input
│   ├── Condensed or full output format
│   └── JSON export option
│
└── naf_categories.yaml
    └── NAF codes + labels + descriptions
```

## Gap Identified: From Data to Decision

Current pipeline gives **administrative listing** (names, stats, contacts) but misses the **enrichment layer** needed to identify interesting companies:

**Missing signals:**
- Tech stack / tools mentioned
- Team composition (engineers, data scientists)
- Mission statement / values alignment
- Recent news / funding / hiring
- Website presence and content

**Solution approach proposed:**
1. **Website discovery** — Add websites from societe.com (SIREN lookup)
2. **Website scraping** — Extract mission, team, tech mentions, job postings
3. **Scoring/ranking** — Rate companies against your criteria (AI/data focus, social impact, growth, team depth)
4. **Curated output** — Ranked list with links + scoring rationale

## Technical Notes

- **Rate limiting:** 150ms delay between requests (7 req/s limit)
- **JSONL format:** Streaming-friendly, parseable line-by-line
- **Extensibility:** YAML config makes adding NAF categories easy
- **Offline work:** Once downloaded, full exploration without API calls

## Decision Point Ahead

For website enrichment, need to decide:
- **Full scraping:** BeautifulSoup + careful crawling of /about, /team, /jobs pages
- **Lightweight:** Just website URLs + manual scoring template
- **API-first:** Use paid services (Crunchbase, LinkedIn API) for structured data

Current recommendation: Start with **societe.com lookup** (free, lightweight) to get URLs, then decide on scraping depth based on initial results.
